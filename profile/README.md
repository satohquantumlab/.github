# satohquantumlab

量子情報・量子コンピューティングに関する研究室の GitHub Organization です。

## リポジトリ命名規則

以下の形式でリポジトリを作成してください。

```
{年度}-{種別}-{人名, 学年}-{内容の短い説明}
```

| 種別 | 用途 | 例 |
|------|------|-----|
| `proj` | 研究プロジェクト | `2026-proj-kamei-medical-qml` |
| `educ` | 授業・教材 | `2026-educ-b3-qiskit-spring` |
| `thesis` | 卒論・修論 | `2026-thesis-yamada-vqe` |

- `proj` と `thesis` には必ず **人名（姓のローマ字）** を含めてください。

- 英数字とハイフンのみ使用（スペース・アンダースコア不可）
- すべて小文字

## リポジトリ作成時のルール

- **公開設定**: 原則 **Private** で作成してください。論文出版やコード公開が決まった段階で管理者と相談のうえ Public に切り替えます。
- **README**: リポジトリには必ず README.md を作成し、プロジェクトの概要・使い方・依存関係を記載してください。
- **`.gitignore`**: 言語やフレームワークに合ったテンプレートを設定してください。特に以下は必ず除外:
  - `.ipynb_checkpoints/`
  - `__pycache__/`
  - 大容量データファイル（データは別途管理）
- **ライセンス**: Public に切り替える際はライセンスファイルを追加してください（迷ったら管理者に相談）。

## 権限について

- メンバーは自分のリポジトリを自由に作成・管理できます。
- 他のメンバーのリポジトリは閲覧のみ可能です。書き込みが必要な場合はリポジトリのオーナーからコラボレーター招待を受けてください。
- 管理者権限が必要な操作（Public への切り替え等）は教員に連絡してください。

## プロジェクト完了・論文出版時

1. コードを整理し、再現可能な状態にする（依存関係の明記、実行手順の記載）
2. 不要なファイルや一時データを削除
3. README を最終版に更新
4. 管理者と相談のうえ Public への切り替え・ライセンス設定を行う

## 卒業時

- 研究に使用したコード・データ処理スクリプトをすべてリポジトリにコミットしてください。
- README に引き継ぎ事項（未完了タスク、既知の問題、環境構築手順など）を記載してください。
- 卒業後のアカウント取り扱いについては別途案内します。
